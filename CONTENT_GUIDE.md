## 1. 파일을 어디서 찾나요?

이 프로젝트의 파일들은 아래 폴더 구조로 정리되어 있습니다.  
**홍보팀이 주로 수정할 파일은 `src/components/` 폴더 안에 있습니다.**

```
CoreDXI-Web/
│
├── src/
│   ├── components/          ⭐ 홍보팀이 주로 수정하는 폴더
│   │   ├── Header.tsx       ← 상단 네비게이션 메뉴 & 버튼
│   │   ├── Hero.tsx         ← 첫 화면(히어로) 섹션
│   │   └── (이후 추가될 컴포넌트들...)
│   │
│   └── app/
│       ├── layout.tsx       ← 브라우저 탭 제목, 공통 설정
│       └── page.tsx         ← 홈페이지 페이지 구성 (컴포넌트 조합)
│
└── public/                  ⭐ 이미지 파일을 여기에 넣습니다
    └── (이미지 파일들을 여기에 추가하세요)
```

### 파일 여는 방법
1. **Cursor IDE**를 사용 중이라면: 왼쪽 파일 탐색기에서 클릭합니다.
2. **Windows 탐색기**를 사용 중이라면: `CoreDXI-Web` → `src` → `components` → 파일을 더블 클릭합니다.

## 2. 헤더 메뉴 항목 수정하기

**수정 파일 경로**: `src/components/Header.tsx`

상단 네비게이션 바의 메뉴 이름과 링크를 수정할 수 있습니다.

### 수정 위치 찾기

파일을 열면 **파일 상단**에 아래와 같은 부분이 있습니다:

```tsx
const NAV_ITEMS = [
  { label: "Home",     href: "/" },
  { label: "회사소개",  href: "/about" },
  { label: "솔루션",   href: "/solutions" },
  { label: "성공사례", href: "/cases" },
  { label: "문의하기", href: "/contact" },
];
```

### 수정 방법

- **메뉴 이름 바꾸기**: `label:` 뒤의 값을 수정합니다.
  - 예) `"회사소개"` → `"About Us"`
- **메뉴 링크 바꾸기**: `href:` 뒤의 값을 수정합니다.
  - 예) `"/about"` → `"/company"`
- **메뉴 순서 바꾸기**: 줄의 순서를 위아래로 이동하면 메뉴 순서도 바뀝니다.
- **메뉴 항목 추가하기**: 같은 형식으로 줄을 추가합니다.
  - 예) `{ label: "채용", href: "/careers" },`
- **메뉴 항목 삭제하기**: 해당 줄 전체를 삭제합니다.

> ⚠️ **주의사항**  
> `{`, `}`, `[`, `]`, `,` 같은 기호는 건드리지 마세요.  
> `label:` 과 `href:` 뒤의 `"따옴표 안 내용"` 만 수정하세요.

## 3. 헤더 버튼 수정하기

**수정 파일 경로**: `src/components/Header.tsx`

헤더 오른쪽에 있는 버튼 2개의 텍스트와 링크를 수정할 수 있습니다.

### 수정 위치 찾기

`NAV_ITEMS` 바로 아래에 아래와 같은 부분이 있습니다:

```tsx
const HEADER_BUTTONS = {
  primary: {
    label: "도입 문의",      // ← 진한 파란색 버튼 텍스트
    href: "/contact",        // ← 진한 파란색 버튼 링크
  },
  outline: {
    label: "상담 신청",      // ← 테두리 버튼 텍스트
    href: "/consult",        // ← 테두리 버튼 링크
  },
};
```

### 수정 방법

- **버튼 텍스트 바꾸기**: `label:` 뒤의 값을 수정합니다.
  - 예) `"도입 문의"` → `"무료 체험"`
- **버튼 링크 바꾸기**: `href:` 뒤의 값을 수정합니다.
  - 예) `"/contact"` → `"https://forms.google.com/..."`

## 4. 메인 화면(히어로) 텍스트 수정하기

**수정 파일 경로**: `src/components/Hero.tsx`

히어로 섹션의 모든 텍스트는 파일 상단의 `HERO_CONTENT` 객체에서 한 번에 수정할 수 있습니다.

### 수정 위치 찾기

파일 상단에서 `HERO_CONTENT`를 찾으세요:

```tsx
const HERO_CONTENT = {
  badge: "AI 기반 AX 전환 솔루션",       // ← 작은 뱃지 문구
  title: "비즈니스의 중심(Core)을\nAI로 깨우다.",  // ← 메인 타이틀 (\n = 줄바꿈)
  subtitle: "복잡한 협업은 심플하게...",   // ← 서브 타이틀
  ...
};
```

### 각 항목 수정 방법

| 항목 | 설명 | 위치 |
|------|------|------|
| `badge` | 타이틀 위 작은 뱃지 문구 | `HERO_CONTENT.badge` |
| `title` | 메인 타이틀 (`\n`으로 줄바꿈) | `HERO_CONTENT.title` |
| `subtitle` | 메인 타이틀 아래 부제목 | `HERO_CONTENT.subtitle` |

> ⚠️ **타이틀에서 줄바꿈 하는 방법**  
> 타이틀 안의 `\n` 은 줄바꿈을 의미합니다.  
> 예) `"첫 번째 줄\n두 번째 줄"` → 화면에서 두 줄로 표시됩니다.

## 5. 히어로 버튼 수정하기

**수정 파일 경로**: `src/components/Hero.tsx`

히어로 섹션의 버튼 2개도 `HERO_CONTENT` 객체에서 수정합니다.

```tsx
const HERO_CONTENT = {
  ...
  primaryCtaText: "솔루션 도입 문의",  // ← 파란색 버튼 텍스트
  primaryCtaHref: "/contact",          // ← 파란색 버튼 링크
  secondaryCtaText: "서비스 소개 보기", // ← 테두리 버튼 텍스트
  secondaryCtaHref: "/about",          // ← 테두리 버튼 링크
  ...
};
```

- **버튼 텍스트 바꾸기**: `primaryCtaText` 또는 `secondaryCtaText` 값을 수정합니다.
- **버튼 링크 바꾸기**: `primaryCtaHref` 또는 `secondaryCtaHref` 값을 수정합니다.
  - 내부 페이지: `"/contact"` 처럼 `/`로 시작
  - 외부 사이트: `"https://calendly.com/coredxi"` 처럼 전체 주소

## 6. 서비스 예시 이미지 교체하기

**수정 파일 경로**: `src/components/Hero.tsx`

히어로 섹션 버튼 아래에 서비스 예시 이미지를 표시할 수 있습니다.  
현재는 점선 플레이스홀더가 표시됩니다.

### 이미지 추가 방법 (3단계)

**1단계**: 사용할 이미지 파일을 `public/` 폴더에 복사합니다.

```
CoreDXI-Web/
└── public/
    └── service-preview.png   ← 여기에 이미지를 넣으세요
```

> 파일 이름 규칙:
> - 한글 사용 금지 → `service-preview.png` (O), `서비스미리보기.png` (X)
> - 공백 사용 금지 → `service-preview.png` (O), `service preview.png` (X)
> - 권장 형식: JPG, PNG, WebP
> - 권장 크기: 1920×1080px, 1MB 이하

**2단계**: `src/components/Hero.tsx` 파일을 열고 `HERO_CONTENT` 안의 `imageSrc` 값을 수정합니다.

수정 전:
```tsx
imageSrc: null,
```

수정 후:
```tsx
imageSrc: "/service-preview.png",
```

> 파일 이름 앞에 `/`(슬래시)를 반드시 붙이세요.

**3단계**: 파일을 저장하면 플레이스홀더 대신 실제 이미지가 표시됩니다.

### 이미지 설명 텍스트 수정

접근성과 SEO를 위한 이미지 설명도 수정하세요:

```tsx
imageAlt: "CoreDXI 서비스 화면 예시",  // ← 이미지를 설명하는 텍스트로 바꾸세요
```

## 7. 일반 이미지 교체하기

**이미지 파일 위치**: `public/` 폴더

### 이미지 교체 방법

1. 새 이미지 파일을 `public/` 폴더 안에 복사합니다.
2. 이미지를 사용하는 컴포넌트 파일에서 파일 경로를 업데이트합니다.

### 이미지 파일 규칙
- 파일 이름에 **한글 사용 금지**
- 파일 이름에 **공백 사용 금지**
- 웹에서 사용할 이미지는 **JPG, PNG, WebP** 형식을 권장합니다.
- 이미지 용량은 **1MB 이하**로 최적화하는 것을 권장합니다.

## 9. 로그인 화면 문구 수정하기

**수정 파일 경로**: `src/app/login/page.tsx`

로그인 화면(`/login`)의 모든 텍스트는 파일 상단의 두 곳에서 관리합니다.

---

### 9-1. 로그인 페이지 기본 문구 수정

파일 상단에서 `LOGIN_CONTENT`를 찾으세요:

```tsx
const LOGIN_CONTENT = {
  title: "Log in to CoreDXI",          // ← 페이지 큰 제목
  emailLabel: "Work email",            // ← 이메일 입력창 라벨
  emailPlaceholder: "name@company.com", // ← 이메일 입력창 예시 텍스트
  continueText: "Continue",            // ← 하단 버튼 텍스트
  signUpText: "Sign up for free",      // ← 우측 상단 회원가입 버튼 텍스트
  signUpHref: "/signup",               // ← 회원가입 버튼 이동 링크
  noAccountText: "Don't have an account?", // ← 하단 안내 문구
};
```

원하는 항목의 값만 수정하면 됩니다.

**예시 — 제목을 한국어로 바꾸고 싶은 경우:**

```tsx
title: "CoreDXI에 로그인",
```

---

### 9-2. 소셜 로그인 버튼 텍스트 수정

파일에서 `SOCIAL_PROVIDERS`를 찾으세요:

```tsx
const SOCIAL_PROVIDERS = [
  { id: "google",  label: "Continue with Google",  ... },  // ← Google 버튼 텍스트
  { id: "slack",   label: "Continue with Slack",   ... },  // ← Slack 버튼 텍스트
  { id: "apple",   label: "Continue with Apple",   ... },  // ← Apple 버튼 텍스트
  { id: "outlook", label: "Continue with Outlook", ... },  // ← Outlook 버튼 텍스트
  { id: "sso",     label: "Continue with SSO",     ... },  // ← SSO 버튼 텍스트
];
```

각 버튼의 `label:` 값을 수정하면 버튼 텍스트가 바뀝니다.

**예시 — 버튼 텍스트를 한국어로 바꾸고 싶은 경우:**

```tsx
{ id: "google", label: "Google로 계속하기", ... },
```

> ⚠️ **주의사항**
> - `id:` 값(예: `"google"`, `"slack"`)은 수정하지 마세요. 내부 식별자입니다.
> - 버튼 순서를 바꾸려면 줄 순서를 이동하세요.
> - 버튼 아이콘 모양 변경이나 버튼 추가/삭제는 개발팀에 문의하세요.

## 11. 수정 후 확인 방법

### 로컬(내 컴퓨터)에서 확인하기

1. 터미널(PowerShell)을 열고, 프로젝트 폴더로 이동합니다.
   ```bash
   cd "C:\Users\SAMSUNG\OneDrive\Desktop\Cursor\CoreDXI-Web"
   ```
2. 개발 서버를 시작합니다.
   ```bash
   npm run dev -- --port 8080 --hostname 127.0.0.1
   ```
3. 브라우저에서 [http://127.0.0.1:8080](http://127.0.0.1:8080) 을 엽니다.
4. 파일을 저장하면 브라우저가 **자동으로** 새로고침됩니다.

### 실제 서버에 반영하기 (배포)

수정 내용을 실제 웹사이트에 반영하려면 **개발팀에 문의**하거나, Vercel에 연결된 GitHub에 코드를 push하면 자동 배포됩니다.

## 10. 관리자 추가 및 권한 관리 방법

**접속 경로**: 브라우저에서 `http://127.0.0.1:8080/admin` (로컬 개발 서버)

> **[중요]** 관리자 페이지는 로그인된 사용자만 접근할 수 있습니다.  
> next-auth 로그인 연동 전 개발 환경에서는 직접 접근이 가능합니다.

---

### 관리자 등록하기

1. 브라우저에서 `/admin/register` 페이지로 이동합니다.
2. 다음 정보를 입력하세요:
   - **이름**: 관리자 이름 (예: 홍길동)
   - **이메일**: 업무용 이메일 (예: hong@coredxi.com)
   - **초기 권한**: 아래 권한 등급 중 선택

3. **등록하기** 버튼을 클릭합니다.
4. 등록 완료 알림이 뜨면 자동으로 관리자 목록 페이지로 이동합니다.

---

### 권한 등급 설명

| 권한 | 영문 | 설명 |
|------|------|------|
| 최고 관리자 | `SUPER_ADMIN` | 모든 기능 접근 + 관리자 등록·삭제 가능 |
| 편집자 | `EDITOR` | 콘텐츠 작성 및 수정 가능 |
| 조회자 | `VIEWER` | 읽기 전용 (기본값) |

> 처음 관리자를 등록할 때는 **VIEWER**로 등록 후, 목록에서 권한을 변경하는 것을 권장합니다.

---

### 권한 변경하기

1. `/admin/users` 페이지에서 관리자 목록을 확인합니다.
2. 권한을 변경할 관리자의 **권한 변경** 열에서 드롭다운을 클릭합니다.
3. 원하는 권한(SUPER_ADMIN / EDITOR / VIEWER)을 선택합니다.
4. 선택 즉시 권한이 변경되며 화면 오른쪽 상단에 알림이 표시됩니다.

---

### 최초 SUPER_ADMIN 계정 만들기

처음 서비스를 시작할 때는 Prisma Studio를 이용해 수동으로 최초 SUPER_ADMIN을 설정합니다:

```bash
# 개발 서버가 실행 중인 터미널에서 새 터미널을 열고 실행
npx prisma studio
```

1. 브라우저에서 `http://localhost:5555` 가 열립니다.
2. **User** 테이블을 클릭합니다.
3. 해당 계정의 `role` 필드를 `SUPER_ADMIN`으로 변경 후 저장합니다.

## 13. 성공사례(포트폴리오) 등록 및 관리 방법

**접속 경로**: 관리자 로그인 후 왼쪽 메뉴 **「성공사례 관리」** (`/admin/portfolio`)

> **[홍보팀]** 코드 파일을 직접 수정하지 않아도, 관리자 화면에서 성공사례를 등록·수정·삭제할 수 있습니다.

---

### 성공사례 목록 보기

1. 관리자 계정으로 `/admin/login` 에 로그인합니다.
2. 왼쪽 사이드바에서 **성공사례 관리**를 클릭합니다.
3. 등록된 사례가 표 제목·고객사·성과 수치·등록일 순으로 표시됩니다.

---

### 새 성공사례 등록하기

1. 목록 화면 오른쪽 상단 **「새 성공사례 등록」** 버튼을 클릭합니다.
2. 아래 항목을 입력합니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| **제목** | 카드·목록에 보이는 대표 제목 | 글로벌 제조사 A사 DX 전환 |
| **고객사명** | 고객사 이름 | (주)OO제조 |
| **성과 수치** | 한 줄 성과 요약 (숫자·% 권장) | 업무 효율 30% 증대 |
| **썸네일 이미지** | 대표 이미지 **URL** (아래 참고) | `https://...` |
| **동영상 URL** | 유튜브·비메오 링크 (선택) | `https://youtube.com/...` |
| **상세 내용** | 프로젝트 배경·솔루션·성과 본문 | 여러 문단 자유 작성 |

3. **저장하기**를 누르면 목록으로 돌아가며 등록이 완료됩니다.

#### 썸네일 이미지 안내

- 화면의 **점선 영역(드래그 앤 드롭)** 은 미리보기용 UI입니다.
- **실제 저장**은 **「이미지 URL」** 입력란에 붙여넣은 주소가 사용됩니다.
- 이미지는 Supabase Storage, CDN, 또는 회사에서 사용하는 이미지 호스팅에 업로드한 뒤, 공개 URL을 복사해 붙여넣으세요.
- URL을 입력하면 아래에 **미리보기**가 표시됩니다.

> 파일을 PC에서 직접 끌어다 놓는 업로드 기능은 준비 중입니다. URL 방식을 사용해 주세요.

---

### 성공사례 수정·삭제하기

- **수정**: 목록에서 해당 행의 **수정** 링크 → 내용 변경 후 **저장하기**
- **삭제**: 목록에서 **삭제** → 확인 창에서 **확인** (복구 불가)

---

### 관리자 메뉴 구조 (참고)

| 메뉴 | 경로 | 용도 |
|------|------|------|
| 대시보드 | `/admin/dashboard` | CMS 홈 (준비 중) |
| 성공사례 관리 | `/admin/portfolio` | 이 가이드의 대상 |
| 시스템 → 관리자 목록 | `/admin/users` | 관리자 계정·권한 |
| 시스템 → 사용자 목록 | `/admin/customers` | 일반 회원 목록 |

### 홈페이지에 자동 반영

- 관리자에서 **저장하기**를 누르면 별도 배포 없이 곧바로 공개 페이지에 반영됩니다.
- 헤더 메뉴 **「성공사례」** → [`/cases`](https://www.coredxi.com/cases) 목록·상세 페이지
- 홈페이지(`/`) 하단 **성공사례** 섹션에는 **최신 3건**이 미리보기로 표시됩니다.
- 수정·삭제도 저장 즉시 `/cases`와 홈에 반영됩니다. 화면이 안 바뀌면 브라우저를 새로고침하세요.

## 12. 자주 묻는 질문

**Q: 파일을 저장했는데 화면이 안 바뀌어요.**  
A: 개발 서버가 실행 중인지 확인하세요. 9번 항목을 따라 서버를 시작하세요.

**Q: 한글을 입력했는데 깨져 보여요.**  
A: 파일을 저장할 때 인코딩을 **UTF-8**로 설정하세요. Cursor IDE와 VS Code는 기본적으로 UTF-8을 사용합니다.

**Q: 실수로 코드를 망가뜨렸어요.**  
A: `Ctrl + Z`로 실행 취소하거나, 개발팀에 원본 복구를 요청하세요. Git으로 버전 관리를 하고 있어 언제든지 복구 가능합니다.

**Q: 메뉴를 클릭했을 때 404 에러가 나요.**  
A: 해당 페이지가 아직 만들어지지 않았습니다. 개발팀에 페이지 생성을 요청하거나, `href` 링크를 임시로 `/` (홈)으로 설정하세요.

**Q: 헤더가 화면 위에 고정되어 있어서 내용이 가려져요.**  
A: 이것은 의도된 디자인입니다. 헤더는 항상 화면 상단에 고정됩니다. 각 섹션의 상단 여백이 자동으로 조정되어 있습니다.

**Q: 새로운 섹션(예: 고객사 로고, 기능 소개)을 추가하고 싶어요.**  
A: 개발팀에 요청하세요. 새 컴포넌트를 만들어 드립니다.

**Q: 어떤 파일을 수정해야 할지 모르겠어요.**  
A: 각 컴포넌트 파일(`Header.tsx`, `Hero.tsx` 등)을 열면 `[홍보팀]` 으로 시작하는 주석이 있습니다. 그 주석 바로 아래의 내용이 수정 대상입니다.
