## 1. 파일을 어디서 찾나요?

이 프로젝트의 파일들은 아래 폴더 구조로 정리되어 있습니다.  
**홍보팀이 주로 수정할 파일은 `src/components/` 폴더 안에 있습니다.**

```
CoreDXI-Web/
│
├── src/
│   ├── components/          ⭐ 홍보팀이 주로 수정하는 폴더
│   │   ├── Hero.tsx         ← 첫 화면(히어로) 섹션
│   │   └── (이후 추가될 컴포넌트들...)
│   │
│   └── app/
│       ├── layout.tsx       ← 브라우저 탭 제목, 공통 설정
│       └── page.tsx         ← 홈페이지 페이지 구성 (컴포넌트 조합)
│
└── public/                  ⭐ 이미지 파일을 여기에 넣습니다
    ├── logo.png             (예시)
    └── hero-image.jpg       (예시)
```

### 파일 여는 방법
1. **Cursor IDE**를 사용 중이라면: 왼쪽 파일 탐색기에서 클릭합니다.
2. **Windows 탐색기**를 사용 중이라면: `CoreDXI-Web` → `src` → `components` → 파일을 더블 클릭합니다.

## 2. 메인 화면(히어로) 텍스트 수정하기

**수정 파일 경로**: `src/components/Hero.tsx`

이 파일을 열면 아래와 같은 **한국어 주석**이 텍스트 위에 달려 있습니다.  
주석 바로 아래의 텍스트만 수정하면 됩니다.

### 메인 타이틀 수정 예시

파일 안에서 아래와 같은 부분을 찾으세요:

```tsx
{/* [홍보팀] 메인 타이틀을 수정하세요 */}
<h1>비즈니스의 중심(Core)을 AI로 깨우다.</h1>
```

`비즈니스의 중심(Core)을 AI로 깨우다.` 부분을 원하는 텍스트로 교체하면 됩니다.

### 서브 타이틀 수정 예시

```tsx
{/* [홍보팀] 서브 타이틀(부제목)을 수정하세요 */}
<p>복잡한 협업은 심플하게, 변화는 단단하게 설계하는 당신의 AX 코어 파트너, CoreDXI</p>
```

`<p>` 와 `</p>` 사이의 텍스트를 수정하면 됩니다.

> ⚠️ **주의사항**  
> `<h1>`, `<p>`, `{/*`, `*/}` 같은 기호들은 건드리지 마세요.  
> **태그 사이의 텍스트만** 수정하면 됩니다.

## 3. 버튼 링크 수정하기

**수정 파일 경로**: `src/components/Hero.tsx`

버튼 링크는 `href=` 부분을 찾아 수정합니다.

### 버튼 링크 수정 예시

파일 안에서 아래와 같은 부분을 찾으세요:

```tsx
{/* [홍보팀] 버튼 클릭 시 이동할 링크를 수정하세요. href="" 안의 값을 바꾸면 됩니다. */}
<a href="/contact">솔루션 도입 문의</a>
```

- **내부 페이지**로 이동하려면: `href="/문의하기"` 처럼 `/`로 시작하는 경로를 입력합니다.
- **외부 사이트**로 이동하려면: `href="https://www.example.com"` 처럼 전체 주소를 입력합니다.
- **버튼 텍스트**를 바꾸려면: `솔루션 도입 문의` 부분만 수정합니다.

## 4. 이미지 교체하기

**이미지 파일 위치**: `public/` 폴더

### 이미지 교체 방법

1. 새 이미지 파일을 `public/` 폴더 안에 복사합니다.
   - 예: `public/hero-background.jpg`
2. 이미지를 사용하는 컴포넌트 파일에서 파일 경로를 업데이트합니다.
   ```tsx
   {/* [홍보팀] 이미지 파일 경로를 수정하세요 */}
   <img src="/hero-background.jpg" alt="이미지 설명" />
   ```
   `src=` 뒤의 경로를 새 파일 이름으로 바꿉니다.

### 이미지 파일 규칙
- 파일 이름에 **한글 사용 금지** (예: `배경이미지.jpg` → `background.jpg`)
- 파일 이름에 **공백 사용 금지** (예: `hero image.jpg` → `hero-image.jpg`)
- 웹에서 사용할 이미지는 **JPG 또는 PNG** 형식을 권장합니다.
- 이미지 용량은 **1MB 이하**로 최적화하는 것을 권장합니다.

## 5. 메타데이터(브라우저 탭 제목) 수정하기

**수정 파일 경로**: `src/app/layout.tsx`

브라우저 탭에 표시되는 제목과 검색 엔진 설명을 수정할 수 있습니다.

```tsx
export const metadata: Metadata = {
  title: "CoreDXI - AI로 비즈니스의 중심을 깨우다",   // ← 탭 제목
  description: "B2B 회의 예약 및 AX 전환 솔루션",       // ← 검색 엔진 설명
};
```

`title`과 `description` 값을 원하는 내용으로 수정합니다.

## 6. 수정 후 확인 방법

### 로컬(내 컴퓨터)에서 확인하기

1. 터미널(명령 프롬프트 또는 PowerShell)을 열고, 프로젝트 폴더로 이동합니다.
   ```bash
   cd "C:\Users\SAMSUNG\OneDrive\Desktop\Cursor\CoreDXI-Web"
   ```
2. 개발 서버를 시작합니다.
   ```bash
   npm run dev
   ```
3. 브라우저에서 [http://localhost:3000](http://localhost:3000) 을 엽니다.
4. 파일을 저장하면 브라우저가 **자동으로** 새로고침됩니다.

### 서버에 반영하기 (배포)

수정 내용을 실제 웹사이트에 반영하려면 **개발팀에 문의**하세요.

## 7. 자주 묻는 질문

**Q: 파일을 저장했는데 화면이 안 바뀌어요.**  
A: 개발 서버(`npm run dev`)가 실행 중인지 확인하세요. 실행 중이 아니라면 위 6번 항목을 따라 시작하세요.

**Q: 한글을 입력했는데 깨져 보여요.**  
A: 파일을 저장할 때 인코딩을 **UTF-8**로 설정하세요. Cursor IDE와 VS Code는 기본적으로 UTF-8을 사용합니다.

**Q: 실수로 코드를 망가뜨렸어요.**  
A: `Ctrl + Z`로 실행 취소하거나, 개발팀에 원본 복구를 요청하세요. Git으로 버전 관리를 하고 있어 언제든지 복구 가능합니다.

**Q: 새로운 섹션(예: 고객사 로고, 기능 소개)을 추가하고 싶어요.**  
A: 개발팀에 요청하세요. 새 컴포넌트를 만들어 드립니다.

**Q: 어떤 파일을 수정해야 할지 모르겠어요.**  
A: 각 컴포넌트 파일(`Hero.tsx` 등)을 열면 `[홍보팀]` 으로 시작하는 주석이 있습니다. 그 주석 바로 아래의 내용이 수정 대상입니다.
