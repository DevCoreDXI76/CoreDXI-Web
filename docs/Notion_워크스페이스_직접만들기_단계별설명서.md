# Notion 워크스페이스 직접 만들기 — 단계별 설명서

이 설명서는 "Vibe Coding Study" 노션 워크스페이스에 7개 표(데이터베이스)를 처음부터 끝까지 직접 만드실 수 있도록 클릭 순서 그대로 정리한 안내서입니다.

진행 상황 메모: "① Projects(프로젝트 목록)" 표는 이미 일부 만들어져 있습니다(이름/상태/날짜 속성까지). 아래 1단계에서 이어서 진행하시면 됩니다.

---

## 시작하기 전에 알아두면 좋은 것

- 표 하나를 만들 때마다 "제목 정하기 → 속성(칸) 추가하기"를 반복합니다.
- 속성을 추가할 때는 표 맨 오른쪽의 **"+"** 버튼을 누르면 됩니다.
- 속성 이름을 입력한 다음, 아래 목록에서 **속성 타입**(텍스트, 선택, 날짜 등)을 클릭해서 골라주시면 됩니다.
- 잘못 만들었다면 속성 이름을 클릭 → "속성 편집"에서 타입이나 이름을 언제든 바꿀 수 있으니 편하게 진행하셔도 됩니다.

---

## 0단계. 최상위 페이지 만들기 (이미 완료됨)

1. 왼쪽 사이드바에서 "새 페이지" 클릭
2. 제목에 `Vibe Coding Study` 입력 후 Enter

이 페이지 안에 아래 7개 표를 전부 순서대로 만들어 넣을 예정입니다.

---

## 1단계. ① Projects (프로젝트 목록) — 이어서 완성하기

이미 "이름 / 상태 / 날짜" 속성까지는 만들어져 있습니다. 아래 속성들을 이어서 추가해주세요.

표 오른쪽 끝의 **"+"** 를 눌러서 하나씩 추가:

| 속성 이름 | 고를 타입 | 비고 |
|---|---|---|
| 시작일 | 날짜 | (이미 "날짜"로 만들어졌다면 이름만 "시작일"로 수정) |
| 종료일 | 날짜 | 새로 추가 |
| 한줄 소개 | 텍스트 | |
| 기술 스택 | 다중 선택 | 옵션 예: React, Python, Supabase (입력하면서 직접 옵션 생성 가능) |
| 사용 라이브러리 | 텍스트 | |
| 사용 스킬/MCP | 텍스트 | |
| 프로젝트 구조 | 텍스트 | |
| 깃허브 repo 링크 | URL | |
| 배포 URL | URL | |

"상태" 속성(이미 만들어짐)의 옵션값으로는 다음을 입력해 주세요: `기획`, `개발중`, `배포완료`, `포트폴리오작성중`, `완료`

---

## 2단계. ② Documents (기획 문서) 만들기

1. ① Projects 표 아래 빈 곳을 클릭
2. `/` 입력 → "표 보기 - 데이터베이스" 또는 "표 보기 - 인라인" 선택
3. 표 제목을 `② Documents 기획문서`로 변경

속성 추가:

| 속성 이름 | 타입 | 옵션/비고 |
|---|---|---|
| 제목 | 텍스트(기본 "이름" 칸을 이걸로 사용) | 예: "PRD v2 - 로그인 플로우 변경" |
| 프로젝트 | 관계형(Relation) | ① Projects 표와 연결 — 만들 때 "어느 데이터베이스와 연결할지" 물어보면 Projects 선택 |
| 문서 종류 | 선택 | 옵션: PRD, 화면설계서, PLAN |
| 버전 | 숫자 | |
| 이전 버전 | 관계형(Relation) | 같은 Documents 표 자기 자신과 연결(Self-relation) |
| 변경 사유 | 텍스트 | |
| 현재 버전 여부 | 체크박스 | |
| 작성일 | 날짜 | |

표를 다 만든 뒤, 표 오른쪽 위의 보기 설정에서 **"갤러리 보기"** 또는 **"보드 보기"**로 바꾸고, "그룹 기준"을 "종류"로 설정하면 카드 형태로 검색하기 편합니다.

---

## 다 만들고 나서 확인할 것

1. 각 표의 "프로젝트" 속성을 클릭해서 ① Projects 표와 잘 연결되어 있는지 확인
2. ① Projects 표에서 프로젝트 하나(예: 테스트용)를 만들어서, 거기 연결된 ②~⑦ 표 항목들이 자동으로 모여 보이는지 확인
3. 보안 주의: ⑦번 표의 "API 키" 종류 항목에는 실제 키 값을 적지 말고, 발급처와 설정 방법만 적기
